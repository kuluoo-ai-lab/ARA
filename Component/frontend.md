INTENT:
"Design, develop, and maintain the Frontend layer of the application to provide an intuitive, responsive, accessible, and performant user interface. The frontend must accurately represent system functionality while ensuring excellent user experience across devices and platforms."

CREATE:
"Create user interfaces, layouts, and interaction flows that translate business requirements into usable visual components, ensuring clarity, consistency, and accessibility."

BUILD:
"Build the frontend using modern frameworks and best practices. Implement reusable components, state management, routing, and client-side validation to deliver a scalable and maintainable UI."

PROCESS:
"Workflow:
   
    STEP-1:
    'Analyze user requirements, user personas, and UX goals.'

    STEP-2:
    'Design wireframes, mockups, and UI flows aligned with brand and usability standards.'

    STEP-3:
    'Select appropriate frontend technologies and frameworks.'

    STEP-4:
    'Develop UI components, layouts, and navigation.'

    STEP-5:
    'Integrate frontend with backend APIs.'

    STEP-6:
    'Implement state management, error handling, and client-side validation.'

    STEP-7:
    'Test responsiveness, accessibility, and cross-browser compatibility.'

    STEP-8:
    'Optimize performance and prepare for deployment.'"

ALTER:
"Allow iterative UI changes based on user feedback and usability testing. All major design changes must be reviewed for UX consistency and performance impact."

MAINTAIN:
"Maintain the frontend through bug fixes, UI improvements, performance optimizations, and framework upgrades while preserving backward compatibility."

RULE:
"Ensure responsive design
Follow accessibility standards (WCAG)
Use reusable components
Minimize client-side load
Maintain visual consistency
Avoid business logic leakage into UI"

MANAGE:
"Manage frontend development by tracking UI tasks, component ownership, design consistency, and integration dependencies."

MONITOR:
"Monitor user behavior, frontend performance metrics, error rates, and usability feedback."

PROTECT:
"Protect the frontend from security risks such as XSS, CSRF, and data leakage by following secure coding and browser security practices."

COLLABORATE:
"Collaborate closely with designers, backend developers, product owners, and QA teams to ensure accurate requirement implementation."

EXECUTE:
"Execute frontend development with a user-first mindset, delivering a polished, secure, and high-performing interface aligned with business goals."
